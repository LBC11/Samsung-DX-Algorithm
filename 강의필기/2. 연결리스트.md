## List

### List의 특성
List: 순서를 가진 데이터의 집합을 가리키는 추상자료형(abstract data type)
1. 동일한 데이터를 가지고 있어도 상관없음
2. 구현방법에 따라 배열을 기반으로 한 순차 List 와 메모리의 동적할당을 기반으로 한 연결 List로 구분한다.

<img src="https://user-images.githubusercontent.com/107410759/215319768-dc313d41-3f79-4f7c-bf9d-fa93fc78f6f0.png" width="800" height="500"/>

- Node: 연결 List에서 하나의 원소에 필요한 데이터를 갖고 있는 자료 단위
    + data field: 원소의 값을 저장하는 부분
    + Link field: 다음 node의 주소를 저장하는 자료구조
- Head: List의 처음 node를 가리키므로 따로 자료가 저장되어있지 않다.

- 삽입 연산
1. 메모리를 할당하여 새로운 노드 생성
2. 노드의 데이터 필드에 정보 저장
3. 삽입될 위치의 바로 앞에 위치한 node의 link field를 new node에 복사
4. new node 의 주소는 앞 node의 link field에 저장

## 이중 연결 List
- 양쪽 방향에서 순회할 수 있도록 node를 연결한 List
- 두 개의 link field와 한 개의 data field로 구성

- 삽입 연산
1. 새로운 new node 생성
2. node의 data field에 정보 저장
3. 선행 node의 next에 new node 정보를 new node의 prev에 선행 node의 주소 저장
4. 후행 node의 prev에 new node 정보를 new node의 next에 선행 node의 주소 저장

- 삭제 연산
1. 삭제할 node의 선행 node의 next에 후행 node의 주소 저장
2. 후행node의 prev에 선행 node의 주소 저장
